Selector

  • *

    • Universal Selector


    권장하지 않음

    브라우저에 과부하 걸림

    1. * {
    2.   color: red;
    3. }
  • #X

    • ID Selector


    유연성 0

    재활용 불가능

    찾을 가능성 거의 없는 요소에 적용하기

    1. #id {
    2.   color: red;
    3. }
  • .X

    • Class Selector


    스타일을 한 그룹의 요소에 적용할 때 사용

    1. .className {
    2.   color: red;
    3. }
  • X

    • Type Selector


    페이지의 모든 요소를 대상으로 할 때

    1. a { color: blue; }
    2. ul { margin-left: 0; }
  • X Y

    • Descendant Selector


    선택자를 사용하여 더 상세히 작업할 때

    선택자가 X Y Z A B.class 처럼 보이면 작업을 잘못하고 있는 것

    1. ul li {
    2.   list-style-type: none;
    3. }
  • X > Y

    • Children Selector


    직계자식만 선택

    성능향상

    자바스크립트 기반 CSS 선택자 엔진으로 추천

    1. .class > p {
    2.   color: red;
    3. }
    1. <section class="class">
    2.   <p> Paragraph </p>
    3.   <ul>
    4.     <p> Paragraph </p>
    5.     <p> Paragraph2 </p>
    6.   </ul>
    7. </section>

    위 예제에선 ul안의 p는 선택되지않음

  • X + Y

    • Adjacent Selector


    앞의 요소 바로 뒤에있는 요소'만' 선택

    1. ul + p {
    2.   color: red;
    3. }

    ul뒤에 오는 첫번째 p만 선택

  • X ~ Y

    • Sibling Selector


    X 아래 모든 Y요소 선택

    1. ul ~ p {
    2.   color: red;
    3. }

    ul 아래있는 모든 p 요소를 선택



Attributes Selector

  • X[title]

    1. a[title] {
    2.   color: yellow;
    3. }

    예제에서 title 속성이 있는 앵커 태그만을 선택

  • X[href="google.com"]

    • 해당 링크와 연결된 전체 anchor태그 선택

    융퉁성 없음

    링크가 www.google.com이면 연결되지않음


    1. a[href="google.com"] {
    2.   color: green;
    3. }
  • X[href*="google.com"]

    • *은 입력값이 속성값 어딘가에 보여야 한다는것을 의미

    www.google.com, net.google.com 등 적용됨

    but, 폭넓은 표현이라 구글로 연결되지않을 가능성농후


    1. a[href*="google.com"] {
    2.   color: green;
    3. }
  • X[href^="http"]

    • http로 시작하는 링크와 연결된 anchor태그 선택

    http://로는 찾아지지않음


    1. a[href^="http"] {
    2.   padding: 5px;
    3. }
  • X[href$=".jpg"]

    • 확장자가 jpg(혹은 .jpg로 끝나는 URL)인 링크와 연결된 anchor태그 선택


    1. a[href$=".jpg"] {
    2.   width: 10px;
    3. }
  • X[data-*="image"]

    • image형식의 파일의 링크와 연결된 anchor태그 선택


    1. a[data-filetype="image"] {
    2.   color: red;
    3. }
    1. <a data-filetype="image">

    위와 같이 anchor태그에 data-filetype속성을 사용



Pseudo Class Selector

    아래 태그들은 순서대로 쓰는것을 권장

  • X:link

    • 모든 링크가 걸린 태그 선택

  • X:visited

    • 방문했던 링크 선택

  • X:hover

    • 마우스를 올렸을 때

  • X:active

    • 마우스를 클릭했을 때

  • X:checked

    • 라디오버튼, 체크박스처럼 체크되는 요소를 대상으로 선택

    1. input[type=radio]:checked {
    2.   border: 1px solid black;
    3. }
  • X:not(selector)

    • Negation Pseudo Class


    특정 태그를 제외 후 선택

    1. body:not(header) {
    2.   color: black;
    3. }
  • X:nth-child(n)

    • n번째 항목 대상 선택

    0부터 시작하지 않음

    1. li:nth-child(3) {
    2.   color: red;
    3. }
    1. li:nth-child(4n) {
    2.   color: red;
    3. }

    위 예제는 li의 4번째 요소마다 선택

  • X:nth-last-child(n)

    • nth-child와 동일하지만 뒤에서부터 셈

    1. li:nth-last-child(2) {
    2.   color: black;
    3. }
  • X:nth-of-type(n)

    • X 요소중 n번째 요소만 선택

    1. ul:nth-of-type(3) {
    2.   border: 1px solid black;
    3. }
  • X:nth-last-of-type(n)

    • X 요소중 뒤에서 n번째 요소만 선택

    1. ul:nth-last-of-type(3) {
    2.   border: 1px solid black;
    3. }
  • X:first-child

    • X 요소중 첫번째 자식 요소만 선택

    목록에서 맨 처음과 맨 나중 항목에서 테두리 선을 제거하는데 이 방식을 흔히 사용

    1. ul li:first-child {
    2.   border-top: none;
    3. }
  • X:last-child

    • X 요소중 마지막 자식 요소만 선택

    목록에서 맨 처음과 맨 나중 항목에서 테두리 선을 제거하는데 이 방식을 흔히 사용

    1. ul > li:last-child {
    2.   color: green;
    3. }
  • X:only-child

    • 단 하나의 자식만을 가진 태그 선택

    거의 사용하지 않음

    1. ul > li:only-child {
    2.   color: red;
    3. }
